# Banking Intent Classification with Unsloth

Fine-tuning **Llama 3.2-3B-Instruct** for intent detection on the [BANKING77](https://huggingface.co/datasets/PolyAI/banking77) dataset using [Unsloth](https://unsloth.ai/) + QLoRA — achieving **91.01% accuracy** on all 77 intent categories.

---

## Project Structure

```
banking-intent-unsloth/
├── scripts/
│   ├── preprocess_data.py    # Data download, preprocessing, ChatML formatting, stratified split
│   ├── train.py              # Fine-tuning pipeline with Unsloth + SFTTrainer + EarlyStopping
│   └── inference.py          # IntentClassification class (__init__ + __call__) + interactive mode
├── configs/
│   ├── train.yaml            # All training hyperparameters
│   └── inference.yaml        # Inference configuration (model path, generation params)
├── sample_data/              # Generated by preprocess_data.py
│   ├── train.csv
│   ├── val.csv
│   ├── test.csv
│   └── label_map.json        # 77 intent labels ↔ numeric IDs
├── train.sh                  # One-command training pipeline
├── inference.sh              # One-command inference demo
├── requirements.txt
└── README.md
```

## Training

### Quick start (recommended)

```bash
bash train.sh
```

This single command runs the full pipeline:
1. **Preprocess** — downloads BANKING77 from GitHub, builds ChatML prompts with all 77 intents, stratified train/val/test split
2. **Fine-tune** — trains Llama 3.2-3B-Instruct with QLoRA (4-bit) + EarlyStopping
3. **Evaluate** — prints accuracy + classification report on the test set
4. **Save** — saves LoRA checkpoint to `model_checkpoint/`

### Resume from checkpoint

```bash
bash train.sh --resume
```

### Run steps individually

```bash
# Step 1: Preprocess only
python scripts/preprocess_data.py --config configs/train.yaml

# Step 2: Train only
python scripts/train.py --config configs/train.yaml

# Step 3: Resume from specific checkpoint
python scripts/train.py --config configs/train.yaml --resume --checkpoint_dir outputs/checkpoint-xxx
```

## Hyperparameters

| Parameter | Value | Notes |
|---|---|---|
| Base model | `unsloth/Llama-3.2-3B-Instruct-bnb-4bit` | 4-bit quantized via Unsloth |
| LoRA rank (r) | 32 | Higher rank = more capacity |
| LoRA alpha | 32 | Equal to r (standard practice) |
| LoRA dropout | 0 | Unsloth-optimized |
| Target modules | q/k/v/o/gate/up/down proj | All projection layers |
| Learning rate | 2e-4 | Cosine decay (Unsloth recommended) |
| Batch size (per device) | 8 | |
| Gradient accumulation | 2 | Effective batch = 16 |
| Epochs | 5 | EarlyStopping patience=2 |
| Max sequence length | 768 | Accommodates 77-intent System Prompt + query |
| Optimizer | AdamW 8-bit | Memory-efficient |
| Warmup ratio | 5% | |
| Weight decay | 0.01 | |
| LR scheduler | Cosine | Smoother convergence |

## Download Trained Model

If you want to skip training and use the pre-trained checkpoint directly:

1. Download the LoRA checkpoint folder from [Google Drive](https://drive.google.com/drive/folders/1flceuRr-QrvimSOZBCfU232GauAju6x8?usp=drive_link) 
2. Place the downloaded folder as `model_best/` inside the project root:
```
banking-intent-unsloth/
├── model_best/              # LoRA adapter checkpoint (194 MB)
│   ├── adapter_config.json
│   ├── adapter_model.safetensors
│   ├── tokenizer.json
│   ├── tokenizer_config.json
│   └── training_meta.json
├── scripts/
...
```
3. Ensure the `checkpoint_path` in `configs/inference.yaml` is set to `"model_best"`.

> **Note:** The base model (`unsloth/Llama-3.2-3B-Instruct-bnb-4bit`, ~2.2 GB) will be automatically downloaded from HuggingFace on the first run and cached in `~/.cache/huggingface/`.

## Inference

### Via shell script

```bash
# Default example
bash inference.sh

# Custom message
bash inference.sh "I lost my credit card"
```

### Via Python

```python
from scripts.inference import IntentClassification

# __init__: loads config, tokenizer, model checkpoint
clf = IntentClassification("configs/inference.yaml")

# __call__: returns predicted intent label
label = clf("I lost my card and need a replacement")
print(label)  # → "lost_or_stolen_card"

label = clf("What is the exchange rate today?")
print(label)  # → "exchange_rate"
```

### Interactive mode (chat)

```bash
python scripts/inference.py --config configs/inference.yaml
```

This opens a live chat session where you can type multiple queries without reloading the model each time.
